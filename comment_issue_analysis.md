# 디시인사이드 댓글 수집 누락 문제 정밀 분석

사용자께서 제공해주신 로그(`0명 (X개 누락)`)와 현재 소스 코드를 대조한 결과, 수집 기능이 완전히 마비된 구체적인 코드상의 결함과 사이트 변경점을 확인했습니다.

## 1. 로그 분석 결과
- **현상**: 게시글 내의 댓글 개수(예: 23개)는 정확히 인식하지만, 정작 수집된 인원은 0명임.
- **의미**: `galleryHelper.js`에서 글 목록을 가져올 때 '댓글 수' 정보는 성공적으로 파싱했으나, `commentHelper.js`에서 해당 글의 실제 댓글 내용을 가져오거나 읽는 단계에서 완전히 실패하고 있음을 의미합니다.

## 2. 코드상 결정적 결함 및 원인

### A. 미니 갤러리 Referer 주소 오류 (버그)
- **파일**: `electron/crawler/commentHelper.js` (L38)
- **코드**: `const referer = "https://m.dcinside.com/board/${gallId}/${gallNum}"`
- **문제**: 미니 갤러리의 경우 주소가 `/board/`가 아닌 `/mini/`여야 합니다. 현재 코드는 모든 갤러리 타입을 `/board/`로 고정하고 있어, 미니 갤러리 수집 시 서버에서 **Referer 불일치**로 요청을 거부하거나 빈 데이터를 보낼 수 있습니다.

### B. HTML 셀렉터 고사 (구조 변경)
- **파일**: `commentHelper.js` (L112-L119)
- **문제**: 현재 코드는 댓글 리스트를 `li.comment-item` 또는 `li` 태그 기반으로 찾습니다.
- **최근 현황**: 디시인사이드는 최근 업데이트를 통해 댓글 리스트를 `li`가 아닌 `div` 태그로 변경했거나, `comment-item` 대신 `cmt_list`와 같은 새로운 클래스명을 사용하고 있습니다. 태그 자체가 리스트가 아닌 계층형 구조로 바뀌면서 기존의 `li` 필터링 로직이 0개를 반환하게 됩니다.

### C. 데이터 추출 포인트 소멸
- **파일**: `commentHelper.js` (L128, L139)
- **문제**: 닉네임과 고유 ID(UID)를 추출할 때 `span.blockInfo`, `span.blockCommentId` 등의 클래스를 찾습니다.
- **최근 현황**: '댓글 개선(멘션 기능)' 도입으로 인해 작성자 정보가 들어있는 클래스명이 대폭 변경되었습니다. 클래스명이 바뀌면 코드는 작성자 정보를 찾지 못해 `nick`이 빈값이 되고, 결과적으로 `authors` 배열에 아무것도 추가되지 않습니다 (L149-L153).

## 3. 해결 방안 (수정 가이드)

### 단계 1: 갤러리 타입에 따른 Referer 동적 생성
- `gallType` 변수를 확인하여 `0, 1`이면 `/board/`, `2`이면 `/mini/`를 사용하도록 수정해야 합니다. (이미 `galleryHelper.js`에는 이 로직이 있으나 `commentHelper.js`에는 누락됨)

### 단계 2: 댓글 리스트 셀렉터 광범위화
- 특정 클래스명에 의존하기보다, 최신 개발자 도구로 확인된 공통 클래스(예: `.comment_list`, `.reply_box`)로 교체하거나 셀렉터를 더 유연하게 확장해야 합니다.

### 단계 3: 작성자 정보 추출 로직 재설계
- `data-name`, `data-info` 속성이 여전히 존재하는지 확인하고, 만약 사라졌다면 텍스트 노드에서 직접 닉네임과 IP/ID를 추출하는 정규식 기반 로직으로의 전환이 필요합니다.

## 4. 요약
현재 모든 댓글이 0명으로 뜨는 것은 **"서버 요청은 보내지만, 받아온 종이(HTML)에서 댓글 정보를 하나도 읽어내지 못하고 있는 상태"**입니다. 이는 100% 디시인사이드의 HTML 구조 개편에 따른 것이며, 코드 내의 **잘못된 Referer 주소**와 **오래된 셀렉터**만 최신화하면 해결될 문제입니다.
